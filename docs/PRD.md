# 產品需求文件 (PRD) - 個人記帳簿

## 1. 專案概述

### 背景與動機
現代人生活節奏快，日常收支繁雜，經常會有「錢都花去哪了」的困擾。為了解決個人財務管理的痛點，建立一個輕量、簡單易用的「個人記帳簿」系統，讓使用者能輕鬆追蹤每日收支，並透過統計數據了解自身的財務狀況，進而達成理財目標。

### 目標用戶
- 想要管理個人財務的學生、上班族、家庭主婦/夫。
- 需要控制每月開銷，避免月底透支的使用者。
- 希望透過簡單工具快速記錄收支，不想使用過於複雜且專業財會軟體的人。

### 核心價值主張
提供直覺且快速的記帳體驗，結合清晰的餘額統計與圖表，幫助使用者輕鬆掌握財務流向，實現聰明理財。

## 2. 功能需求

### 主要功能清單
1. **紀錄收入與支出**：新增、編輯、刪除單筆收支紀錄（包含日期、金額、分類、備註）。
2. **分類收入與支出**：將收支歸類（如：飲食、交通、娛樂、薪水等），並支援自訂分類。
3. **計算每月的收入與支出**：自動加總每月收支，並顯示當前餘額。
4. **視覺化報表顯示**：透過圓餅圖或長條圖顯示每月各分類的收支佔比與趨勢。
5. **預算設定與提醒**：可設定每月的總支出預算，當支出接近或超過預算時在介面上給予視覺提示。

### 使用者故事 (User Stories)
- **紀錄收入與支出**：作為一名 **記帳使用者**，我希望 **能夠快速新增一筆支出或收入紀錄**，以便 **我能隨時追蹤我的金錢流向**。
- **分類收入與支出**：作為一名 **記帳使用者**，我希望 **能在記帳時選擇或新增收支分類**，以便 **我能清楚知道錢花在哪些種類上**。
- **計算每月收支**：作為一名 **記帳使用者**，我希望 **系統能自動結算每月的總收入、總支出與結餘**，以便 **我能快速了解當月的財務健康狀況**。
- **視覺化報表顯示**：作為一名 **記帳使用者**，我希望 **能看到依分類圓餅圖顯示的支出比例**，以便 **我能一眼看出最大的開銷來源，進而調整消費習慣**。
- **預算設定與提醒**：作為一名 **記帳使用者**，我希望 **能設定每月的花費預算，並在首頁看到預算達成率**，以便 **提醒自己控制開銷，避免超支**。

## 3. 非功能需求

### 技術限制
- **後端框架**：Flask (Python)
- **模板引擎**：Jinja2
- **資料庫**：SQLite (適合個人單機或輕量級應用)
- **前端技術**：HTML, CSS (可用 Vanilla CSS 或輕量框架), JavaScript (用於圖表繪製，如 Chart.js)

### 效能與安全考量
- **效能**：系統操作需具備高回應速度，頁面載入時間與新增紀錄的反應時間應極短，提供流暢的使用體驗。
- **安全**：
  - 防止常見的網頁攻擊（如 SQL Injection 與 XSS）。
  - （選配）若有使用者系統，需實作基本的使用者驗證與密碼加密，保護個人財務資料。

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必須具備 - 最小可行性產品)
- 紀錄收入與支出（新增、檢視、編輯、刪除）。
- 內建基本的收支分類選單（如：飲食、交通、購物、薪水）。
- 首頁顯示當月總收入、總支出與目前餘額。
- 收支明細列表（支援按月份瀏覽）。

### Should Have (應該具備 - 重要優化)
- 自訂收支分類（新增或刪除分類）。
- 簡單的圖表呈現（如圓餅圖顯示當月支出比例）。

### Nice to Have (有了更好 - 未來擴充功能)
- 每月預算設定與超支警告進度條。
- 資料匯出功能（如匯出成 CSV/Excel）。
- 長條圖顯示過去數個月的收支趨勢。
